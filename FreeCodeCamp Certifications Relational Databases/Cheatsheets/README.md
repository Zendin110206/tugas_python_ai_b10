# Relational Databases Cheatsheets

This folder stores quick lookup references for syntax learned in the
freeCodeCamp Relational Databases Certification track.

The detailed workshop notes and `*_Syntax_Reference.md` files remain in their
original topic folders. This folder is different: it is meant to be opened when
I need to quickly remember how to write a command, query, condition, loop, join,
or import pattern.

## Files

| File | Focus | Best used when |
| --- | --- | --- |
| `Bash_Cheatsheet.md` | Bash terminal, scripting, streams, text processing, and menu automation syntax. | Writing shell scripts, loops, conditions, input handling, redirection, pipes, `grep`, `sed`, `diff`, CSV parsing, `psql` automation, and interactive terminal menus. |
| `Git_Cheatsheet.md` | Git repository, branch, merge, rebase, stash, reset, revert, and ignore-file workflow. | Checking repository state, staging commits, organizing feature branches, resolving conflicts, squashing commits, and protecting local `.env` files. |
| `Nano_Cheatsheet.md` | Nano terminal editor shortcuts and file editing workflow. | Editing files directly in a terminal, saving with `Ctrl+O`, exiting with `Ctrl+X`, cutting and pasting lines, and handling small Bash output edge cases while editing. |
| `PostgreSQL_SQL_Cheatsheet.md` | PostgreSQL and SQL syntax. | Creating databases, designing tables, querying rows, filtering with `WHERE`, grouping, joining tables, updating state, and exporting dumps. |

## Scope Policy

These cheatsheets are based on material already practiced in this repository.
When new freeCodeCamp Relational Databases topics are completed, new syntax can
be added here without changing the older workshop notes.

## How to Use

1. Open the topic file that matches the problem.
2. Find the section by goal, not by theory.
3. Copy the syntax pattern.
4. Replace table names, column names, variables, or values with the current
   problem context.

Example:

```text
Need to filter rows by text?
Open PostgreSQL_SQL_Cheatsheet.md -> WHERE Conditions -> Text filters.
```
